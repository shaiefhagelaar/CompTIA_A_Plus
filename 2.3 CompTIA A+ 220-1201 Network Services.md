# CompTIA A+ 220-1201: 2.3 Network Services

This summary covers the essential network services and protocols required for Objective 2.3 of the CompTIA A+ Core 1 exam. These services are the "glue" that allows devices to communicate, find each other, and exchange data.

---

## 1. Domain Name System (DNS)
DNS is often called the "phonebook of the internet." It translates human-readable domain names (e.g., `google.com`) into IP addresses (e.g., `142.250.190.46`).

* **Port:** UDP/53 (Queries) and TCP/53 (Zone transfers).
* **Hierarchy:** Operates in a tree-like structure (Root -> TLD -> Authoritative).
* **Key Records:**
    * **A:** Maps a hostname to an IPv4 address.
    * **AAAA:** Maps a hostname to an IPv6 address.
    * **MX (Mail Exchanger):** Identifies the server responsible for receiving email.
    * **CNAME (Canonical Name):** An alias of one name to another (e.g., `www.example.com` to `example.com`).
    * **TXT (Text):** Used for human-readable notes, SPF, and DKIM (security).

---

## 2. Dynamic Host Configuration Protocol (DHCP)
DHCP automates the process of assigning IP addresses, subnet masks, default gateways, and DNS server info to devices on a network.

* **Ports:** UDP/67 (Server) and UDP/68 (Client).
* **The DORA Process:**
    1.  **D**iscover: Client looks for a server.
    2.  **O**ffer: Server offers an address.
    3.  **R**equest: Client "claims" the offer.
    4.  **A**cknowledge: Server confirms the lease.
* **Reservations:** You can configure the DHCP server to always give the same IP to a specific MAC address (ideal for printers).

## 3. Web and File Services
These protocols allow for the retrieval of web pages and the transfer of files between systems.

| Protocol | Port | Description |

| **HTTP** | 80 | Hypertext Transfer Protocol. Unencrypted web traffic. |
| **HTTPS** | 443 | HTTP Secure. Encrypted via SSL/TLS. |
| **FTP** | 20/21 | File Transfer Protocol. 20 (Data), 21 (Control). Unencrypted. |
| **SFTP** | 22 | Secure FTP. Uses SSH for encryption. |
| **TFTP** | 69 | Trivial FTP. Simple, no authentication; used for booting diskless workstations. |

## 4. Email Protocols
Understanding how mail is sent versus how it is retrieved is critical for troubleshooting client configurations.

* **SMTP (Simple Mail Transfer Protocol):** Used to **send** mail from a client to a server or between servers. 
    * Port: TCP/25 (Legacy) or TCP/587 (Modern secure).
* **POP3 (Post Office Protocol v3):** Downloads mail to a local device and typically deletes it from the server. 
    * Port: TCP/110 (Unencrypted) or TCP/995 (Secure).
* **IMAP4 (Internet Message Access Protocol v4):** Syncs mail across multiple devices; mail stays on the server.
    * Port: TCP/143 (Unencrypted) or TCP/993 (Secure).

---

## 5. Remote Access & Management
These protocols allow technicians to manage devices from a distance.

### Remote Desktop Protocol (RDP)
* **Port:** TCP/3389.
* **Function:** Provides a full graphical user interface (GUI) to a remote Windows machine.

### SSH (Secure Shell) vs. Telnet
* **SSH (Port 22):** Encrypted terminal access. This is the industry standard for secure remote management.
* **Telnet (Port 23):** Unencrypted terminal access. Avoid using this, as passwords are sent in cleartext (like shouting your password in a crowded room).

### SNMP (Simple Network Management Protocol)
* **Port:** UDP/161 (Queries) and UDP/162 (Traps).
* **Function:** Used to gather statistics and monitor the "health" of network devices (routers, switches, etc.).

---

## 6. Infrastructure Services
* **SLP (Service Location Protocol):** TCP/UDP 427. Used by devices to find services (like printers) on a local network without prior configuration.
* **SMB/CIFS (Server Message Block):** TCP/445. Windows standard for file and printer sharing. 
* **AFP (Apple Filing Protocol):** TCP/548. The Mac equivalent of SMB (though modern macOS supports SMB).
* **LDAP (Lightweight Directory Access Protocol):** TCP/389. Used to query and update a central database of users and computers (e.g., Active Directory).
